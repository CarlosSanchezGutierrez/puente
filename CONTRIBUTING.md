# Contribuir a Puente Impacto

Gracias por tu inter&eacute;s en contribuir.

Puente Impacto trabaja con una l&oacute;gica de tecnolog&iacute;a social, documentaci&oacute;n, educaci&oacute;n e impacto comunitario. Las contribuciones deben mantener un enfoque profesional, claro y responsable.

## Criterios generales

- Mantener claridad de alcance.
- Evitar cambios innecesariamente complejos.
- Cuidar privacidad y datos sensibles.
- Documentar decisiones relevantes.
- Priorizar soluciones mantenibles.
- Validar cambios antes de proponerlos.

## Flujo sugerido

1. Revisar el contexto del proyecto.
2. Crear una rama descriptiva.
3. Realizar cambios peque&ntilde;os y enfocados.
4. Ejecutar validaciones locales.
5. Abrir un pull request con descripci&oacute;n clara.

## Validaciones

Antes de abrir un pull request:

```bash
pnpm --filter web lint
pnpm --filter web build
```

## Estilo de comunicaci&oacute;n

El tono de Puente Impacto debe ser profesional, sobrio, claro y nada pretencioso.

Evitar lenguaje exagerado como:

- Revolucionamos el mundo.
- Somos l&iacute;deres.
- Impacto garantizado.
- La soluci&oacute;n definitiva.

Usar lenguaje pr&aacute;ctico:

- Apoyamos.
- Documentamos.
- Definimos alcance.
- Damos seguimiento.
- Construimos herramientas &uacute;tiles.
